**Background**

Conventional droplet size analysis often reduces the distribution to a few summary metrics. It is common to report percentile diameters such as d10, d50, d90 (diameters below which 10%, 50%, 90% of the volume is contained).
While these d-values (or moments like the Sauter mean diameter) provide a convenient overview, they do not fully describe the distribution’s shape. 
Different emulsions can share the same median droplet size (d50) yet have very different high-end tail proportions (e.g. presence of a few large droplets in one sample but not another), which would impact stability differently.

**Challenges**
1. _Label inefficiency_: Labelled data generation requires computationally and temporally expensive experiments and simulations. So need a model that works in low-labelled data regimes.

**Task**
Full particle size distribution prediction. Basically we formulate the problem as a classification for regression task where the regression labels of particle size quantiles are binned into a finite set of values between the maximum and minimum particle sizes possible and a random forest regressor is used for classification of each quantile into one of the bins.

For the input features (mixing conditions) corresponding to each quantile data point, we included both the experimental factors and an identifier of the quantile level. Each quantile of each sample was treated as a separate training instance with the following features.

Movement pattern as a categorical feature indicating which of the three trajectory patterns (Hexagon 1, 2, or 3) was used. This was one-hot encoded into three binary dummy variables for the model. Sampling portion as a categorical feature for the sample’s spatial position (Top, Middle, Bottom). Similarly one-hot encoded (or integer 1–3) for inclusion in the feature vector. Mixing time as a continuous feature (in seconds). This was normalized (scaled to zero-mean and unit-variance) before training so that the range of values (short vs long mixing) would be treated uniformly by the model. Mixing speed as a continuous feature. This was also normalized prior to modeling.Quantile index as an ordinal feature indicating which quantile is being predicted. Rather than having 100 separate outputs, we give the model a clue as to whether this data point corresponds to, say, the 5th percentile or the 95th percentile of the distribution. We encoded this as a continuous variable p (ranging from 0.01 to 0.99 for 1%–99%) or as an index 1–100. In practice, providing the cumulative probability value p for the quantile proved effective, as it allows the model to learn different functional relationships for different parts of the distribution (for instance, the features might interact differently to predict the tail vs the median).

We trained a Random Forest classifier (using the scikit-learn implementation). The RF was configured with 300 decision trees (“estimators”) and using Gini impurity as the splitting criterion. We enabled class_weight='balanced' to account for the fact that some quantile size bins (especially those corresponding to extreme small or large sizes) are less frequent in the training data than others – this helps mitigate any bias where the classifier might otherwise always predict the dominant classes. Other hyperparameters of the RF were left at defaults (max features for split ≈ sqrt(number of features), no maximum tree depth set, etc.), as initial tests indicated robust performance without intensive tuning. The training process involved feeding the 8100 training instances to the RF, which builds an ensemble of decision trees that partition the feature space and attempt to classify each quantile into the correct size bin
